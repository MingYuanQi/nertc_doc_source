<!--keywords: 质量监测,质量数据回调,网络状态,统计数据 -->
在通话场景中，开发者经常需要了解当前通话的通话质量、设备状态等信息，监测通话的整体体验；也可将部分质量数据在 UI 层面展示给用户，使用户能够及时了解当前通话的整体质量。NERTC SDK 支持将关键的音视频状况、网络状况、设备状态的相关指标实时回调给 APP 应用层，应用层可以将收到的数据进行展示或统计。

例如，在进行多人连麦时，您可以展示用户的实时网络质量，如下图所示。

<img style="width:400px" src=" https://yx-web-nosdn.netease.im/common/bc9cb52f00e1948491c5666d6db5c5c0/网络显示.png " alt="文本">

## <span id="功能介绍">功能介绍</span>

NERTC SDK 提供注册质量监测观测器 [`NERtcStatsObserver`](https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtcCallback.NERtcStatsObserver.html) 的接口与相关回调方法，支持在通话中实时同步并返回质量数据，包括上下行网络质量、本地通话统计信息、本地和远端的音视频流统计信息等。

## <span id="示例项目">示例项目</span>

网易云信提供 [MediaStats 示例项目源码](https://github.com/netease-im/Basic-Video-Call/tree/master/Group-Video/NERtcSample-GroupVideoCall-Harmony-Typescript/entry/src/main/ets)，您可以参考该源码实现通话中质量监测。

## <span id="实现方法">实现方法</span>

调用 [`setStatsObserver`](https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtc.NERtc.html#setstatsobserver) 方法注册质量观测器，主动设置相应回调，回调名称与返回的质量数据信息的对应关系如下表所示。

| 回调 | 统计信息 |
| -- | -- |
| <a href="https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtcCallback.NERtcStatsObserver.html#onNetworkQuality" target="_blank">`onNetworkQuality`</a> | [上下行网络质量同步](#上下行网络质量同步) |
| <a href="https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtcCallback.NERtcStatsObserver.html#onRtcStats" target="_blank">`onRtcStats`</a> | [统计信息同步](#统计信息同步) |
| <a href="https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtcCallback.NERtcStatsObserver.html#onLocalAudioStats" target="_blank">`onLocalAudioStats`</a>、<a href="https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtcCallback.NERtcStatsObserver.html#onRemoteAudioStats" target="_blank">`onRemoteAudioStats`</a> | [音频质量同步](#音频质量同步) |
| <a href="https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtcCallback.NERtcStatsObserver.html#onLocalVideoStats" target="_blank">`onLocalVideoStats`</a>、<a href="https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtcCallback.NERtcStatsObserver.html#onRemoteVideoStats" target="_blank">`onRemoteVideoStats`</a> | [视频质量同步](#视频质量同步) |

::: note note
若您需要关闭质量监测功能，再次调用此方法并传入 `null` 即可。
:::

**示例代码**如下：

``` typescript
    NERtcSDK.getInstance().setStatsObserver({

      //返回本地本地通话统计信息
      onRtcStats(stats: NERtcConstants.NERtcStats): void {

      },
      //返回本地设备发送音频的统计信息
      onLocalAudioStats(stats: NERtcConstants.NERtcAudioSendStats): void {

      },
      //返回通话中每个远端用户音频流的统计信息
      onRemoteAudioStats(statsArray: Array<NERtcConstants.NERtcAudioRecvStats>): void {

      },
      //返回本地设备发送视频流的统计信息
      onLocalVideoStats(stats: NERtcConstants.NERtcVideoSendStats): void {

      },
      //返回通话中每个远端用户视频流的统计信息
      onRemoteVideoStats(statsArray: Array<NERtcConstants.NERtcVideoRecvStats>): void {

      },
      //返回通话中每个成员的上下行网络质量
      onNetworkQuality(statsArray: Array<NERtcConstants.NERtcNetworkQualityInfo>): void {

      }
    })
```

## 相关回调

### <span id="上下行网络质量同步">上下行网络质量同步</span>

[`onNetworkQuality`](https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/interfaces/NERtcCallback.NERtcStatsObserver.html#onnetworkquality) 回调以数组的形式向您同步当前通话中每个成员的上下行网络质量。

- 上行网络质量打分基于实际发送码率、上行网络丢包率、平均往返时延和上行网络抖动计算。
- 下行网络质量打分基于下行网络丢包率、平均往返时延和下行网络抖动计算。

    ::: note note
    - 每隔 2 秒您会收到房间内所有用户的网络质量同步。
    - 实际发送码率与目标发送码率的比值越高，该网络下的通话质量越好，该网络质量打分越高。
    :::

[`NERtcNetworkQualityInfo`](https://doc.yunxin.163.com/docs/interface/nertc/harmonyos/typedoc/Latest/zh/classes/NERtcConstants.NERtcConstants.NERtcNetworkQualityInfo.html) 数组中各参数的说明如下表所示。

<table>
<tr>
    <th width="30%">参数</th>
    <th width="70%">参数说明</th>
</tr>
    <tr>
    <td>uid</td>
    <td>用户 ID，指定是哪个用户的网络质量统计。</td>
</tr>
    <tr>
    <td>upStatus</td>
    <td>该用户的上行网络质量。</td>
</tr>

    <td>该用户的下行网络质量。</td>
